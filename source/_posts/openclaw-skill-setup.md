# 【AI助手】给 OpenClaw 装上了几个有用的 Skill

昨天把 OpenClaw 装好了，今天继续折腾——给它装了几个 skill，顺手做了一件一直想做但懒得做的事。

---

## 先说说今天装的这几个 Skill

> OpenClaw 有一个 skill 市场叫 **ClawHub**，里面各种各样的小工具。装之前记得先跑一遍安全审查，别啥都往里塞。

### 🔒 skill-vetter — 装 Skill 前的守门员

任何 skill 安装之前先让它扫一遍，评估风险等级再决定装不装。毕竟是跑在自己电脑上的东西，小心点没坏处。

这个 skill 审查很严格，会检查代码里有没有：
- 🚨 往外部服务器发送数据
- 🚨 读写敏感文件（SSH 配置、API Key 等）
- 🚨 执行来路不明的命令

**⚠️ 必须装。**

---

### ✏️ humanizer — 去掉文章里的 AI 味

昨天写的那篇 OpenClaw 安装教程，回头一看，浓浓的 AI 味——

> "说起来挺好笑的"、"说实话"出现了两次、"整个安装过程其实不算复杂"……都是 AI 写作的典型毛病。

用 humanizer skill 扫了一遍，改掉了所有过度修饰的表达，读起来顺畅多了。

---

### 📝 self-improving-agent — 记录错误，持续进化

这个挺有意思的：它会记录我每次犯的错误、用户的纠正、以及学到的新东西，存成 `.learnings/` 目录下的 Markdown 文件。

> 说白了就是给自己建一个**错题本**，下次再遇到类似问题能快速想起来。

---

### 🔍 multi-search-engine — 17 个搜索引擎聚合

把国内外的搜索引擎整合在一起，Baidu、Google、DuckDuckGo、Brave、Sogou……要什么有什么。不需要 API key，直接调用。

| 国内引擎 | 国际引擎 |
|---------|---------|
| 百度 | Google |
| 搜狗 | DuckDuckGo |
| 360 | Brave |
| 必应 | Yahoo |

## 🎯 用 AI 整理了一份芒市美食地图

四月打算去德宏芒市过泼水节，在网上找到一张当地美食清单的图片，列了二十多家店，看着就让人发愁要怎么规划路线。

### 整体流程

```
图片 → AI 识别文字 → 搜索坐标 → 整理 Excel → 批量导入地图
```

### 第一步：用 AI 识别图片里的文字

OpenClaw 内置了图片识别能力，直接把图片扔给它，让它列出图中所有店名。AI 读图能力比 OCR 强多了，印刷字体、手写体混在一起也能基本认出来。

识别出了 **27 个点位**，分类如下：

| 分类 | 代表店铺 |
|------|---------|
| 🍜 早餐 | 银家早点、烂路早点、咩萝卜早点 |
| 🌙 小吃/夜市 | 新玩厂夜市、勐巴娜西美食城 |
| 🛕 景点 | 勐焕大金塔、勐焕银塔 |
| 🍖 傣味/烧烤 | 线恋烧烤、罗记小肉串、芒杏岩兴烧烤 |
| ☕ 缅味 | 蜜玛餐饮、缅故里、蔓德里 |
| 🦀 活螃蟹 | 天然活蟹、七只虾铺 |

### 第二步：搜索每个店的坐标

有了店名之后，用搜索引擎逐个搜地址。大多数店都能搜到具体坐标，有些搜不到的就根据所在区域估算。

> 纯手工做的话估计要蛮久，交给 AI 并行处理快很多。

### 第三步：Apple 地图？走不通

因为用的 macOS，一上来自然想到 Apple 地图。KML（Keyhole Markup Language）是地图标注的行业标准格式，理论上双击就能打开。

结果 macOS 版 Apple 地图对 KML 的支持烂得离谱——**根本不支持直接导入 KML 文件**。

试了各种方法：

| 方法 | 结果 |
|------|------|
| 拖拽到 Apple 地图窗口 | ❌ 没用 |
| File → Import 菜单 | ❌ macOS 版压根没有这个选项 |
| 搜索攻略 | ❌ 全是针对 iOS 的，macOS 不适用 |

换思路。

### 第四步：高德云图批量导入

最后选了高德地图网页版 **高德云图**（wia.amap.com），支持批量导入 POI 数据。

但又踩了一个坑——**必须按高德给的模板格式才能识别**，不支持通用 KML。

之前自己生成的 CSV 导入失败，原因是经纬度顺序搞反了，而且格式不符合模板要求。

正确姿势：

1. 打开高德云图，创建新地图
2. 点「批量导入」→「下载模板」
3. 按模板填入：**名称、经度、纬度**（⚠️ 经度在前、纬度在后）
4. 保存为 `.xlsx` 格式，上传

最终生成了 `芒市美食之旅_高德导入.xlsx`，27 个点位全部正确显示在高德云图里。

### 最终效果

去芒市之前可以先在地图上规划路线，到了现场再根据实际情况调整。**从拿到图片到生成可用地图，大概花了不到半小时。**

## 📈 装了股票行情查询 CLI

本来想用 tavily（能实时抓取网络数据的 skill），逛 ClawHub 的时候发现了 `tecent-finance`，可以通过腾讯财经公开 API 直接调数据，不需要任何 key。

```bash
tfin price AAPL TSLA NVDA        # 美股
tfin price sh601611 sh600519      # A股
tfin price hk09988 hk00700        # 港股
tfin compare AAPL MSFT GOOGL     # 多股对比
```

实测可用，查阿里巴巴（HK09988）和中国核建（SH601611）都拿到了实时行情。

---

## ⏰ 接上了 macOS 提醒事项

这是今天最有实用价值的一个操作——用 AppleScript 写了一个纯 Python CLI，可以直接操作 macOS 原生的"提醒事项" App：

```bash
remind add "卖半仓核建" --due tomorrow --list 股票
remind today         # 今日待办
remind overdue       # 逾期事项
```

设了一个明天早上 9:20 的待办，到时候系统会弹通知提醒。
